# Agent Hospital: A Simulacrum of Hospital with Evolvable Medical Agents

**Authors:** Junkai Li, Yunghwei Lai, Weitao Li, Jingyi Ren, Meng Zhang, Xinhui Kang, Siyu Wang, Peng Li, Ya-Qin Zhang, Weizhi Ma, Yang Liu
**Affiliations:** Tsinghua University (AIR + CS Department)
**Date:** 2024-05-05 (revised 2025-01-17)
**Paper:** [PDF](https://arxiv.org/abs/2405.02957)

---

## TL;DR

Agent Hospital is a virtual hospital where all patients, nurses, and doctors are LLM-powered agents. Doctor agents evolve by treating tens of thousands of synthetic patient agents (generated by coupling LLMs with medical knowledge bases), accumulating a medical case base (learning from successes) and experience base (learning from failures via reflection). After evolution, the doctor agents achieve **92.2% on MedQA (USMLE)**, outperforming MedAgents, CoT, and Medprompt — all without using any labeled training data.

---

## Key Novel Ideas

### 1. SEAL — Simulacrum-based Evolutionary Agent Learning

The paper's core paradigm, combining AlphaGo Zero's self-play philosophy with Smallville's agent simulation:

**Two components:**
- **Simulacrum Construction:** Build a virtual world that generates unlimited training data. The LLM + medical knowledge base produces diverse, realistic patient cases (demographics, symptoms, exam results) without human annotation.
- **Agent Evolution:** Let doctor agents improve by treating simulated patients, storing successful cases and reflecting on failures.

**Why this matters:** Traditional medical AI requires expensive labeled data (human-annotated diagnoses). SEAL generates its own training signal through the closed loop of treatment → outcome → feedback. The virtual world runs orders of magnitude faster than real time, so a doctor agent can "treat" 50,000 patients in hours rather than a career.

### 2. Dual Memory Architecture — Medical Case Base + Experience Base

Doctor agents maintain two distinct knowledge stores:

**Medical Case Base (learning from success):**
- Stores successful treatment cases as Q&A pairs
- Separate sub-bases for each task (exam selection, diagnosis, treatment)
- Retrieved via RAG (cosine similarity with text-embedding-ada-002) at inference time
- Also populated by "reading medical books" — converting knowledge base documents into Q&A format

**Experience Base (learning from failure):**
- When a doctor agent makes a wrong diagnosis, it compares its answer with the ground truth and generates a natural-language principle/rule
- **Experience Validation:** Each principle is tested against exemplar Q&A pairs from medical documents. Only principles that lead to correct diagnoses on new cases are retained.
- **Experience Refinement:** Principles are reformatted for consistency and re-validated
- This is essentially tuning-free rule accumulation — the agent builds up clinical heuristics through practice

**Why dual memory works:** Case base gives the agent concrete precedents (like a doctor remembering similar patients). Experience base gives the agent abstract rules (like clinical guidelines learned from mistakes). Together they mirror how real physicians develop expertise.

### 3. Automatic Patient Agent Generation

The critical enabler for unlimited training data. Given a disease, the system generates a complete patient in 5 sequential steps:

1. **Disease Selection** — Choose from 339 diseases across 32 departments
2. **Basic Information** — Name, gender, age (informed by disease epidemiology via knowledge base)
3. **Medical History** — Prior conditions relevant to the disease
4. **Symptoms** — Generated from disease knowledge, diverse and realistic
5. **Medical Examination Reports** — Lab results, imaging findings, etc., grounded in medical knowledge

A **quality control agent** validates that generated patient data is medically consistent with the knowledge base. This prevents the LLM from hallucinating unrealistic clinical presentations.

### 4. Closed-Loop Treatment Cycle with Feedback

Unlike real hospitals where patient outcomes are often unknown to doctors, Agent Hospital provides complete feedback:

8 events form the cycle: Disease Onset → Triage → Registration → Consultation → Medical Examination → Diagnosis → Medicine Dispensary → Convalescence

If the patient doesn't recover, they return for another treatment cycle. The doctor gets explicit signal about whether their treatment worked, enabling continuous learning.

|---|---|
| Base LLM (doctor agents) | GPT-3.5 (evolution), GPT-4o (MedQA evaluation) |
| Departments | 32 total (21 clinical + 11 non-clinical) |
| Diseases covered | 339 |
| Training patients per department | Up to 20,000 |
| Doctor agents | 42 |
| Nurse agents | 4 |
| Medical knowledge source | Baidu Health Encyclopedia + DXY |
| RAG embeddings | text-embedding-ada-002 (OpenAI) |
| Evaluation tasks | Medical exam selection, diagnosis, treatment recommendation |
| Test set per department | 200 patient agents |
| Simulation environment | Tiled (map editor) + Phaser (web game framework) |

## Training Pipeline

### Phase 1: Knowledge Base Construction
- Scrape disease information from Baidu Health Encyclopedia and DXY
- Extract symptoms, examination results, treatment plans for 339 diseases
- Organize by clinical department

### Phase 2: Patient Agent Generation
- For each disease, generate hundreds of diverse patient agents using LLM + knowledge base
- Quality control agent validates medical consistency
- Generate up to 20,000 patients per department

### Phase 3: Doctor Agent Evolution (MedAgent-Zero)
- Doctor agents start with only the base LLM's knowledge (no labeled data)
- Treat patient agents through the full clinical cycle
- **Success path:** Correct cases stored in medical case base
- **Failure path:** Wrong answers trigger reflection → generate principle → validate → store in experience base
- Doctors also "read medical books" during off-hours (knowledge base → Q&A conversion → case base)
- Continue treating patients; accuracy improves with log-like scaling

### Phase 4: Real-World Evaluation
- Apply evolved doctor agents to MedQA (USMLE) benchmark
- Same case base and experience base used; no MedQA training data used

## Key Results

### MedQA (USMLE) — Real-World Benchmark (GPT-4o base)

| Method | Accuracy |
|---|---|
| Base (GPT-4o) | 88.2% |
| MedAgents | 89.2% |
| CoT | 90.4% |
| Medprompt | 91.5% |
| **Agent Hospital (Ours)** | **92.2%** |

All methods use GPT-4o. Agent Hospital uses no MedQA training data.

### Virtual World Diagnostic Accuracy — Scaling with Patients Treated

| Patients Treated | Respiratory Dept Accuracy (Virtual) | Respiratory Dept Accuracy (MedQA) |
|---|---|---|
| 0 | ~66% | ~85% |
| 10,000 | ~93% | ~92% |
| 20,000 | ~96% | ~95% |
| 50,000 | ~98% | ~97% |

### Per-Disease Improvement (Examples from Cardiology)

| Disease | Before Evolution | After Evolution |
|---|---|---|
| Rheumatic Heart Disease | 9% | 82% |
| Hypertension | 67% | 96% |
| Myocarditis | 67% | 83% |
| Arrhythmia | 50% | 90% |

### Virtual-to-Real Transfer

Virtual world accuracy and MedQA accuracy improve in tandem (both curves rise with more patients treated), confirming that skills learned in simulation transfer to real-world medical questions.

## Key Takeaways

1. **Simulacra can replace labeled datasets for domain-specific agent training.** SEAL generates its own training data by coupling LLMs with domain knowledge bases, eliminating the need for human annotation. This is the paper's most generalizable contribution — the paradigm applies to any domain with structured knowledge (law, finance, education).

2. **Doctor agents show clear scaling laws in evolution.** Diagnostic accuracy improves monotonically with the number of patients treated, following a log-like curve: fast improvement in the first 10,000 cases, then gradual refinement. This mirrors human learning curves and suggests predictable performance gains from more simulation.

3. **Learning from failure (experience base) is as important as learning from success (case base).** The reflection mechanism that generates principles from wrong answers and validates them against new cases is what enables the doctor agent to avoid repeating mistakes — a capability that pure case retrieval lacks.

4. **Virtual world skills transfer to real-world benchmarks without domain-specific fine-tuning.** The 92.2% MedQA accuracy using only virtual-world experience and no USMLE training data demonstrates genuine knowledge transfer. Virtual and real accuracy curves are correlated.

5. **The closed feedback loop is an unfair advantage over real hospitals.** In real medicine, doctors rarely get systematic feedback on treatment outcomes. Agent Hospital provides immediate, complete feedback for every case, enabling much faster learning than real clinical experience.

6. **Quality control of synthetic patients is critical.** Without the validation agent checking that generated patient data is medically consistent, the training signal would be noisy. This is an underappreciated design choice that prevents the doctor agent from learning from unrealistic cases.

7. **Experience accumulation decelerates over time, like human learning.** The rate of new validated experiences slows as the doctor agent treats more patients — early cases yield the most novel principles, and later cases increasingly overlap with existing knowledge.

8. **The base LLM is frozen — all improvement comes from the external memory.** Unlike fine-tuning approaches, SEAL doesn't modify model weights. All evolution happens in the case base and experience base (RAG). This means the approach is model-agnostic and can benefit from better base models without retraining.

9. **Limitations are honest and significant:** The base model is frozen (can't learn fundamentally new reasoning), treatment plans are limited to recommendations (no procedural skills), and there's no cross-department consultation. These mirror real limitations of text-only AI in medicine.

10. **The AlphaGo Zero analogy is apt but incomplete.** AlphaGo Zero had a perfect game simulator with well-defined rules. Medicine has ambiguity, comorbidities, and subjective judgment. The quality of the simulacrum (how well synthetic patients mirror real patients) is the binding constraint on this approach.
